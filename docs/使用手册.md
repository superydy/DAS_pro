# DAS_pro 使用手册

ETH-5520 分布式声波传感（DAS）板卡上位机软件

---

## 1. 软件是做什么的

ETH-5520 是一块双通道、500MSps 的 DAS 数据采集解调板卡：激光向一根普通
光纤打脉冲，光纤沿线任何位置的振动都会改变该处回波的相位，板卡把回波采
集并解调后通过千兆网口（TCP）发给电脑。效果上，一根几公里的光纤变成一排
连续分布的"麦克风"。

本软件（DAS_pro.exe）是跑在电脑上的控制端，负责四件事：

1. **配置板卡** —— 下发触发频率、采样点数、解调参数等全部设置；
2. **接收数据流** —— 持续接收板卡上传的数据（可达几十 MB/s）；
3. **实时显示** —— 波形、频谱、空间/时间两种视图、幅度监测；
4. **录制数据** —— 原始数据存 `.bin`，并自动生成记录全部参数的同名 `.json`。

## 2. 运行软件

### 2.1 环境要求

- 64 位 Windows 10 / 11；
- 不需要安装 Python 或任何运行库，单个 exe 即可运行；
- 不要放在 `C:\Program Files` 等系统目录（录数据需要写权限）。

### 2.2 两种启动方式

| 方式 | 操作 | 用途 |
|---|---|---|
| 演示模式 | 在 exe 所在文件夹打开 cmd，运行 `DAS_pro.exe --simulator` | 无板卡时演示/学习，软件内置一个假板卡（模拟器），IP 自动填 127.0.0.1 |
| 正常模式 | 直接双击 `DAS_pro.exe` | 连接真实板卡 |

首次运行的正常现象：

- 启动需等待几秒（单文件 exe 解包）；
- Windows SmartScreen 弹蓝色警告 → 点"更多信息"→"仍要运行"（软件未购买
  数字签名证书所致，并非病毒）；
- 防火墙询问是否允许联网 → 必须点"允许"，否则连不上板卡。

### 2.3 连接真实板卡前的网络准备

1. 网线将电脑与板卡直连（或接同一交换机）；
2. 电脑 IP 手动设为与板卡同网段：板卡出厂默认 `192.168.1.100`，
   则电脑设为 `192.168.1.x`（x 取 1~254 中不与板卡冲突的值，如 10）；
3. 软件底部"板卡地址"填板卡 IP，端口固定 5000。

> **同网段铁律**：电脑 IP 与板卡 IP 前三段必须相同、第四段必须不同，
> 否则无法通信。连接失败时第一个检查这里。

## 3. 界面说明

界面为全中文。中部是三张大图，右侧是一列功能按钮和两个常驻面板，
底部是板卡地址与开始/退出按钮。各组参数收在按钮弹窗里，点开修改、
确定生效，主界面保持干净。

### 3.1 顶部工具条

| 控件 | 含义 |
|---|---|
| 每包扫描数 | 板卡攒够 N 次脉冲数据打成一包上传（原 FrameNum） |
| 保存数据 | 全量录制开关，**必须在开始采集之前勾选**（原 SaveData） |
| 显示通道 | 四通道模式下选择显示哪对通道（0-1 或 2-3） |
| 网络速率 | 当前参数下的网络数据率（预估值） |
| 空间视图 | 空间/时间视图切换（仅相位解调模式可用） |
| 观察位置 | 空间模式下锁定观察的位置点序号 |

### 3.2 采集参数（右栏"采集参数…"按钮）

| 参数 | 默认值 | 含义 |
|---|---|---|
| 触发频率 | 2000 Hz | 每秒向光纤发多少个激光脉冲，等于每个位置振动信号的采样率 |
| 脉冲宽度 | 100 ns | 影响空间分辨率（100ns ≈ 10m） |
| 总采样点数 | 5120 | 每个脉冲沿途采集的点数，决定监测光纤长度；必须是 16 的倍数 |
| 旁路点数 | 2 | 丢弃每次扫描开头的无效点数 |
| 数据源 | 原始波形 | 上传类型：原始波形 / IQ / 反正切&幅值 / **相位解调（实际监测用）** |
| 上传速率 | 500MSps | 采样率 |
| 上传通道数 | 双通道 | 上传通道数（单/双/四） |
| 光纤长度 | （计算值） | 可监测光纤长度 = 点数 × 0.2m（500MSps 时） |
| 中心频率 | 80 MHz | 解调中心频率，对应光路移频器件（AOM）频率 |
| 触发方向 | 输出 | 输出=板卡发触发（当主控）；输入=接收外部触发 |
| 时钟源 | 板载晶振 | 板载晶振 / 外部 10MHz 参考（多机同步用） |
| 相位位宽 | 32位 | 相位数据位宽；16位省一半带宽、精度略低 |
| 上传抽取比 | 1 | 相位上传抽取比：N=4 即每 4 个脉冲传 1 次 |

采集运行中弹窗变为只读，需先停止采集再修改。

### 3.3 相位解调（右栏"相位解调…"按钮，仅数据源=相位解调时生效）

| 参数 | 默认值 | 含义 |
|---|---|---|
| 空间平均阶数 | 25 | 空间滑动平均阶数（压噪声） |
| 空间合并点数 | 25 | 每 N 个点合成 1 个输出位置（5120/25=204 个位置） |
| 区域差分阶数 | 2 | 相当于标距，影响灵敏度 |
| 去趋势滤波带宽 | 20 Hz | 滤掉温漂等慢变化 |
| 偏振分集 | 关 | 抑制偏振衰落 |
| 解调数据率 | 250M | 进入相位解调器的数据率 |

> 注：本组参数的精确物理定义请以厂家《ETH-5520 用户手册》为准。

### 3.4 中部 · 三张图

- **图1（黄）**：通道 0 波形。横轴为点序号，原始波形模式下即对应光纤位置
  （0=板卡端，最大值=光纤远端）；纵轴为 12 位采集幅度（约 ±2048）；
- **图2（白）**：通道 1 波形；勾选"频谱显示"后变为通道 0 的频谱；
- **图3 · 光纤回波强度（幅度监测）**：光纤沿线回波强度，用于发现信号过弱
  的区段和判断光纤实际终点。仅相位解调模式下板卡附带上传。

### 3.5 右栏

- **帧信息（板卡上报，只读）** —— 每包数据的"回执"，用于核对收发一致：
  设备标识、数据类型（0/2/3/4/5，应与数据源对应）、每包扫描数、
  每扫描点数、实收点数（=扫描数×点数）、累计收包；
- **频谱** —— "频谱显示"把图2切为频谱；"功率谱密度"把单位从 dBm 换成
  dBm/Hz；
- **单点监测/音频…** —— 见 3.7 节；
- **区域监测…** —— 见 3.8 节；
- **修改板卡IP…** —— 把板卡自身 IP 永久改为弹窗里填的值。改完后板卡需
  重新上电，底部连接地址同步改为新值，建议贴标签记录；
- **数字输出…** —— 控制板卡数字 IO 引脚：使能位 + 输出电平，确定即发送。

### 3.6 底部 · 连接栏

- **板卡地址** —— 要连接的**板卡当前 IP**（不是电脑 IP），演示模式自动填
  127.0.0.1；端口固定 5000；
- **绿色圆点** —— 亮 = TCP 已连接；
- **开始采集** —— 一键完成连接 → 下发全部参数 → 开始采集；再点变"停止采集"；
- **退出** —— 退出软件。

> **两个 IP 不要混**：底部"板卡地址"是去**连接**板卡（拨对方号码）；
> 右栏"修改板卡IP…"是**改掉**板卡自己的地址（给对方换号码）。
> 电脑自身的 IP 在 Windows 网络设置里配置，软件中不需要填。

### 3.7 单点监测 / 音频窗口

数据源选**相位解调**后，点右栏"单点监测/音频…"弹出独立窗口，回答
"这一个点振成什么样"：

- **自动识别振动点**：第一张图为振动强度分布（横轴=光纤位置），红虚线
  为自动阈值（全纤中位数 × 阈值系数，默认 6 倍）。超阈值即判定振动，
  顶部标红"⚠ 检测到振动：位置 N"。勾选"自动跟踪振动点"则监测位置自动
  跳到该点；
- **范围**两个框限定检测区间，终点务必设在光纤实际终点以内（参考图3
  跌落位置），否则会被末端噪声干扰；
- 第二、三张图为监测点的**时域波形**（最近 8 秒）和**频谱**；
- **播放声音**：监测点振动实时从扬声器播出，敲光纤能听见；
- **单点录制**：只录监测位置一个点（约 8 KB/s，可长录）。自选保存位置，
  格式 WAV 音频 / CSV 文本 / 二进制 float32，均带同名 .json 参数档案；
  录制期间位置锁定，一个文件对应一个光纤位置。

### 3.8 区域监测窗口（瀑布图）

点右栏"区域监测…"弹出，回答"范围内**哪些点**在振、**什么时候**振"：

- **多点检测**：范围内所有超阈值的位置全部列出（相邻位置归并为一个
  振动点），顶部显示"⚠ 检测到 N 个振动点：…"；
- **瀑布图（核心）**：横轴=光纤位置，纵轴=时间（新数据在下），颜色亮度
  =振动强度。哪里振动哪里亮：持续振动是一条竖向亮带，沿光纤移动的
  扰动是一条斜线，多点同时振动一眼全看到；
- **振动点列表**：点击任一项，第三张图显示该点的时域波形；
- **区域录制**：保存范围内**全部位置**的数据（二维数组 float32 + .json
  档案，json 内附 numpy 读取示例）。范围在开始录制时锁定。

> 演示模式下模拟器在光纤 1/3 处制造一个 50 Hz 振动点，两个监测窗口
> 都能直接看到效果。

## 4. 标准操作流程

### 4.1 演示（无板卡）

1. cmd 运行 `DAS_pro.exe --simulator`；
2. 参数保持默认，点 **开始采集** → 绿灯亮、累计收包增长、波形滚动；
3. 勾"频谱显示"看频谱；采集参数里数据源切"相位解调" + 勾"空间视图" +
   调"观察位置"体验空间模式；再打开"单点监测/音频…"和"区域监测…"
   体验自动识别振动点和瀑布图；
4. 录数据：**停止采集 → 勾"保存数据" → 开始采集 → 数秒 → 停止采集**，
   底部状态栏显示文件完整路径；
5. **退出**。

### 4.2 连接真实板卡

1. 按 2.3 节完成网络准备；
2. 双击启动，底部填板卡 IP；
3. 在"采集参数…"里按光路实际情况设参数（中心频率对应移频器频率、
   总采样点数按光纤长度 ÷ 0.2m 取整到 16 的倍数）；
4. 开始采集；先用"原始波形"确认回波正常，再切"相位解调"监测；
5. 长时间录制请用相位解调模式（数据率远低于原始波形），或用监测窗口
   的单点/区域录制（数据率更低）。

## 5. 数据文件说明

勾选"保存数据"录制后，在 **exe 同目录的 `save_data` 文件夹**生成一对
文件（单点/区域录制的保存位置则由你自己选择）：

```
1-15-24-04_D.bin    ← 原始数据（序号1，15:24:04 开始录制）
1-15-24-04_D.json   ← 同名参数档案
```

- 文件名格式：`序号-时-分-秒_D`，每次重新开始采集序号加一；
- `.bin` 为板卡上传字节的原样存储（比特级精确，无任何转换）；
- `.json` 记录录制时的全部参数、起止时间、接收帧数，并附 numpy 读取示例：

```python
import numpy as np
data = np.fromfile("1-15-24-04_D.bin", dtype="<i2")   # dtype 见 json
data = data.reshape(-1, 5120, 2)                       # (扫描, 点, 通道)
```

**关于文件大小**：原始波形模式数据率很高（如 39 MB/s，即每小时约 137GB），
录十几秒就是数百 MB，属正常现象而非错误。日常监测请用相位解调模式，
或用单点/区域录制。

## 6. 常见问题

| 现象 | 原因与处理 |
|---|---|
| 双击 exe 弹蓝色警告 | SmartScreen 对未签名软件的提示，点"更多信息→仍要运行" |
| 杀毒软件报毒 | PyInstaller 打包程序的常见误报，添加信任 |
| 启动慢几秒 | 单文件 exe 每次启动需自解包，正常 |
| 点"开始采集"报"启动失败" | ① 板卡 IP 填错；② 电脑与板卡不同网段；③ 防火墙未放行；④ 网线/电源问题 |
| 录制后找不到文件 | ① "保存数据"须在开始采集前勾选；② 文件在 exe 同目录 save_data 下，状态栏显示完整路径 |
| bin 文件巨大 | 原始波形数据率本身就高，见第 5 节；长录用相位解调或单点/区域录制 |
| 图3（幅度监测）空白 | 仅相位解调模式且板卡上传监测数据时有内容；模拟器不产生 |
| 帧信息不能编辑 | 设计如此：它是板卡上报的回执（只读），可设置的"每包扫描数"在顶部工具条 |
