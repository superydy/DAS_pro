# DAS_pro 使用手册

ETH-5520 分布式声波传感（DAS）板卡上位机软件

---

## 1. 软件是做什么的

ETH-5520 是一块双通道、500MSps 的 DAS 数据采集解调板卡：激光向一根普通
光纤打脉冲，光纤沿线任何位置的振动都会改变该处回波的相位，板卡把回波采
集并解调后通过千兆网口（TCP）发给电脑。效果上，一根几公里的光纤变成一排
连续分布的"麦克风"。

本软件（DAS_pro.exe）是跑在电脑上的控制端，负责四件事：

1. **配置板卡** —— 下发触发频率、采样点数、解调参数等全部设置；
2. **接收数据流** —— 持续接收板卡上传的数据（可达几十 MB/s）；
3. **实时显示** —— 波形、频谱、空间/时间两种视图、幅度监测；
4. **录制数据** —— 原始数据存 `.bin`，并自动生成记录全部参数的同名 `.json`。

---

## 2. 运行软件

### 2.1 环境要求

- 64 位 Windows 10 / 11；
- 不需要安装 Python 或任何运行库，单个 exe 即可运行；
- 不要放在 `C:\Program Files` 等系统目录（录数据需要写权限）。

### 2.2 两种启动方式

| 方式 | 操作 | 用途 |
|---|---|---|
| 演示模式 | 在 exe 所在文件夹打开 cmd，运行 `DAS_pro.exe --simulator` | 无板卡时演示/学习，软件内置一个假板卡（模拟器），IP 自动填 127.0.0.1 |
| 正常模式 | 直接双击 `DAS_pro.exe` | 连接真实板卡 |

首次运行的正常现象：

- 启动需等待几秒（单文件 exe 解包）；
- Windows SmartScreen 弹蓝色警告 → 点"更多信息"→"仍要运行"（软件未购买
  数字签名证书所致，并非病毒）；
- 防火墙询问是否允许联网 → 必须点"允许"，否则连不上板卡。

### 2.3 连接真实板卡前的网络准备

1. 网线将电脑与板卡直连（或接同一交换机）；
2. 电脑 IP 手动设为与板卡同网段：板卡出厂默认 `192.168.1.100`，
   则电脑设为 `192.168.1.x`（x 取 1~254 中不与板卡冲突的值，如 10）；
3. 软件左下角"板卡地址"填板卡 IP，端口固定 5000。

> **同网段铁律**：电脑 IP 与板卡 IP 前三段必须相同、第四段必须不同，
> 否则无法通信。连接失败时第一个检查这里。

## 3. 界面说明

界面分左、中、右三栏。

### 3.1 左栏 · 采集参数（下发给板卡）

| 参数 | 默认值 | 含义 |
|---|---|---|
| TrigFreq | 2000 Hz | 触发频率：每秒向光纤发多少个激光脉冲，等于每个位置振动信号的采样率 |
| TrigWidth | 100 ns | 脉冲宽度，影响空间分辨率（100ns ≈ 10m） |
| TotalPointNum | 5120 | 每个脉冲沿途采集的点数，决定监测光纤长度；必须是 16 的倍数 |
| BypassPointNum | 2 | 丢弃每次扫描开头的无效点数 |
| DataSrc | RawData | 上传数据类型：RawData 原始波形 / IQ / Arctan&Sqrt 中间量 / **Phase 解调相位（实际监测用）** |
| UploadRate | 500MSps | 采样率 |
| UploadChNum | Two | 上传通道数（1/2/4） |
| FiberLen | （只读） | 软件算出的可监测光纤长度 = 点数 × 0.2m（500MSps 时） |
| CenterFreq | 80 MHz | 解调中心频率，对应光路移频器件（AOM）频率 |
| TrigDir | OUT | OUT=板卡发触发（当主控）；IN=接收外部触发 |
| ClkSrc | OnBoard | 时钟源：板载晶振 / 外部 10MHz 参考（多机同步用） |
| PhaseBit | 32Bit | 相位数据位宽；16Bit 省一半带宽、精度略低 |
| TrigFreqDecRatio | 1 | 相位上传抽取比：N=4 即每 4 个脉冲传 1 次 |

### 3.2 左栏 · 相位解调（仅 DataSrc=Phase 时生效）

| 参数 | 默认值 | 含义 |
|---|---|---|
| SpaceAvgOrder | 25 | 空间滑动平均阶数（压噪声） |
| SpaceMergePoints | 25 | 空间合并：每 N 个点合成 1 个输出位置（5120/25=204 个位置） |
| RegionDiffOrder | 2 | 区域差分阶数（相当于标距，影响灵敏度） |
| DetrendFilterBW | 20 Hz | 去趋势滤波带宽（滤掉温漂等慢变化） |
| Polarization | DIS | 偏振分集开关（抑制偏振衰落） |
| Rate2PhaseDem | 250M | 进入相位解调器的数据率 |
| AudioEN | （灰） | 振动音频播放，后续版本提供 |

> 注：本组参数的精确物理定义请以厂家《ETH-5520 用户手册》为准。

### 3.3 左栏 · 板卡地址与按钮

- **四个数字框** —— 要连接的**板卡当前 IP**（不是电脑 IP）。演示模式自动
  填 127.0.0.1（指本机模拟器）；
- **PortNum:5000** —— 端口固定；
- **绿色圆点** —— 亮 = TCP 已连接；
- **START** —— 一键完成连接 → 下发全部参数 → 开始采集；再点变 STOP；
- **QUIT** —— 退出软件。

### 3.4 中栏 · 顶部工具条

| 控件 | 含义 |
|---|---|
| FrameNum | 每包扫描数：板卡攒够 N 次脉冲数据打成一包上传 |
| SaveData | 录制开关，**必须在 START 之前勾选** |
| DisplayIndex | 四通道模式下选择显示哪对通道（01 或 23） |
| ETH_Throught | 当前参数下的网络数据率（预估值） |
| Space | 空间/时间视图切换（仅 Phase 模式可用） |
| RegionIndex | 空间模式下锁定观察的位置点序号 |

### 3.5 中栏 · 三张图

- **图 1（黄）**：通道 0 波形。横轴为点序号，RawData 模式下即对应光纤位置
  （0=板卡端，最大值=光纤远端）；纵轴为 12 位采集幅度（约 ±2048）；
- **图 2（白）**：通道 1 波形；勾选 SpectrumEn 后变为通道 0 的频谱；
- **AMP Monitor**：光纤沿线回波强度监测，用于发现信号过弱的区段。仅当板
  卡上传监测数据（Phase 模式附带）时有内容；模拟器不产生此类数据，演示
  模式下为空属正常。

### 3.6 右栏

- **帧信息（只读）** —— 板卡每包数据的"回执"，用于核对收发一致：
  - Identifier：21792 = 0x5520，设备标识；
  - DataType：0/2/3/4/5，当前包的数据类型，应与 DataSrc 对应；
  - FrameNum / PointNumPerScan：板卡确认的每包扫描数、每扫描点数；
  - ReadPointsNum：本包每通道实收点数（= 扫描数 × 点数）；
  - RecvFrameCnt：开始采集以来累计收到的包数。
- **频谱开关**：SpectrumEn 将图 2 切换为频谱；PSD EN 把单位从 dBm（功率谱）
  换成 dBm/Hz（功率谱密度）。
- **ConfUserIP** —— **修改板卡自身的 IP**（永久生效）。仅在板卡需要换网段时
  使用；改完后左下角连接地址需同步改为新值。
- **数字输出** —— 控制板卡数字 IO 引脚：DOBitEN 选择使能哪些位，DOBit 设
  电平，SetDO 发送。可用于联动外部设备。

### 3.7 两个 IP 地址的区别（易混淆）

| 位置 | 名称 | 作用 | 比喻 |
|---|---|---|---|
| 左下角 | 板卡地址 | 软件**去连接**的板卡当前 IP | 拨打对方的电话号码 |
| 右上角 | ConfUserIP | **把板卡自己的 IP 改成**填入的值 | 给对方换一个新号码 |

电脑自身的 IP 在 Windows 网络设置里配置，软件中不需要填写。

## 4. 标准操作流程

### 4.1 演示（无板卡）

1. cmd 运行 `DAS_pro.exe --simulator`；
2. 参数保持默认，点 **START** → 绿灯亮、RecvFrameCnt 增长、波形滚动；
3. 勾 SpectrumEn 看频谱；DataSrc 切 Phase + 勾 Space + 调 RegionIndex
   体验空间模式；
4. 录数据：**STOP → 勾 SaveData → START → 数秒 → STOP**，底部状态栏显示
   文件完整路径；
5. **QUIT** 退出。

### 4.2 连接真实板卡

1. 按 2.3 节完成网络准备；
2. 双击启动，左下角填板卡 IP；
3. 按光路实际情况设参数（CenterFreq 对应移频器频率、TotalPointNum 按
   光纤长度 ÷ 0.2m 取整到 16 的倍数）；
4. START 采集；先用 RawData 确认回波正常，再切 Phase 监测；
5. 长时间录制请用 Phase 模式（数据率远低于 RawData）。

## 5. 数据文件说明

勾选 SaveData 录制后，在 **exe 同目录的 `save_data` 文件夹**生成一对文件：

```
1-15-24-04_D.bin    ← 原始数据（序号1，15:24:04 开始录制）
1-15-24-04_D.json   ← 同名参数档案
```

- 文件名格式：`序号-时-分-秒_D`，每次重新 START 序号加一；
- `.bin` 为板卡上传字节的原样存储（比特级精确，无任何转换）；
- `.json` 记录录制时的全部参数、起止时间、接收帧数，并附 numpy 读取示例：

```python
import numpy as np
data = np.fromfile("1-15-24-04_D.bin", dtype="<i2")   # dtype 见 json
data = data.reshape(-1, 5120, 2)                       # (扫描, 点, 通道)
```

**关于文件大小**：RawData 模式数据率很高（如 39 MB/s，即每小时约 137GB），
录十几秒就是数百 MB，属正常现象而非错误。日常监测请录 Phase 模式。

## 6. 常见问题

| 现象 | 原因与处理 |
|---|---|
| 双击 exe 弹蓝色警告 | SmartScreen 对未签名软件的提示，点"更多信息→仍要运行" |
| 杀毒软件报毒 | PyInstaller 打包程序的常见误报，添加信任 |
| 启动慢几秒 | 单文件 exe 每次启动需自解包，正常 |
| 点 START 报"启动失败" | ① 板卡 IP 填错；② 电脑与板卡不同网段；③ 防火墙未放行；④ 网线/电源问题 |
| 录制后找不到文件 | ① SaveData 须在 START 前勾选；② 文件在 exe 同目录 save_data 下，状态栏显示完整路径 |
| bin 文件巨大 | RawData 数据率本身就高，见第 5 节；长录用 Phase |
| AMP Monitor 空白 | 仅 Phase 模式且板卡上传监测数据时有内容；模拟器不产生 |
| 帧信息不能编辑 | 设计如此：它是板卡上报的回执（只读），可设置的 FrameNum 在顶部工具条 |
